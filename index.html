Amazon Reviews: Predicting Ratings

This project analyzes Amazon U.S. customer reviews to predict review scores (star_rating) using PySpark for scalable data processing and machine learning. Due to the scale of the dataset, distributed computing techniques were required to efficiently process and analyze the data. The objective is to predict review ratings from textual and metadata features, enabling practical applications such as automated sentiment analysis and product quality monitoring.

Star Rating Distribution

Insights

The distribution of our target variable, star_rating, is highly skewed, with the majority of reviews concentrated in the 4- and 5-star range as lower ratings are relatively underrpresented. This imbalance highlights the need for resampling strategies, class weighting, and proper feature selection.

Challenges

Other computational and analytical challenges beyond the highly skewed ratings involve storage and processing demands, limited data processing resources, and appropriately handling a dataset that includes high-dimensional text and sparsely populated categories. These characteristics require appropriate feature engineering, data balancing, and algorithm selection to achieve accurate predictions.

Data Preprocessing

Missing Values

Reviews missing review_body were excluded, as they provide no textual signal for predicting ratings. Only reviews from the U.S. marketplace were retained to maintain English-language consistency. Missing values are extremely rare (<0.02%), so simple deletion or deterministic imputation is sufficient.

Data Imbalance

Some product categories have different counts, and the target variable star_rating is skewed toward 4–5 stars. We use Spark to reduce computation overhead. If imbalance affects modeling, class weighting or resampling techniques are applied.

Feature Transformations

  • Helpfulness Ratio: helpful_votes / total_votes with safeguards for zero votes.
  • Review Length: Numerical feature capturing the length of review_body.
  • Product Category: One-hot encoded for modeling.
  • All features combined into a single vector for ML models.

Feature Importance Analysis

Text-based features dominate prediction performance: review_word_counts and review_len are the most important. This shows that more detailed reviews provide stronger signals for star ratings. In contrast, verified_purchase_idx has negligible impact, meaning verified purchases do not strongly influence predictions.

Model 2: Random Forest w/ PCA + K-Means Clustering

RF with Standard Features

The left-side matrix shows a strong bias toward predicting 5-star ratings, with a large concentration of predictions in the final column. While the model correctly classifies many true 5-star reviews, it frequently misclassifies ratings 1–4 as 5, particularly for classes 3 and 4 where the majority of predictions are pushed upward. In other words, this pattern indicates that the model is heavily influenced by class imbalance as it captures high-rating behavior well, but performs poorly on lower ratings; this suggests limited ability to learn meaningful distinctions across the full rating spectrum.

RF with PCA Features

Using PCA-transformed features shows a complete collapse in predictive performance, with the model predicting only 5-star ratings for every instance. All observations, regardless of their true rating, are classified as 5, resulting in zero correct predictions for classes 1–4 and only correct predictions for true 5-star ratings.

Conclusion

Category Observation Impact / Notes
True Positives Correctly predicted 5-star reviews High-rating reviews are well captured; model reflects common patterns.
True Negatives Correctly predicted low ratings (1–3) Limited coverage; lower ratings are harder to predict accurately.
False Positives Predicted high rating (5) but actual was lower Bias toward 5-star ratings due to class imbalance; affects precision for lower ratings.
False Negatives Predicted lower rating but actual was 5 Rare, but indicates the model sometimes underestimates highly positive reviews.

Precision vs. Recall: The model exemplifies a high recall for 5-star ratings, successfully identifying the majority of highly positive reviews. However, at the expense at reduced precision for lower-rated classes, the model also frequently misclassifies ratings 1 through 4 due to class imbalance. From a predictive performance perspective, this trade-off indicates that the model prioritizes sensitivity to the dominant class over the accurate differentiation of less frequent rating levels. Consequently, while the model is effective at capturing positive sentiment, its predictive reliability for intermediate or negative ratings is limited. Future work could mitigate this skew through techniques such as class weighting, resampling, or incorporating additional discriminative features to enhance the balance between precision and recall across all rating categories.
